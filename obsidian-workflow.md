# Obsidian Vault 操作工作流程

## 一、概述

本文档定义了 AI 操作 Obsidian Vault 的标准工作流程，确保数据安全和操作可控。

## 二、权限验证流程

每次操作前，必须执行权限检查：

```
1. 读取 ~/ObsidianVault/AI-Memory/obsidian-rules.yaml
2. 确认目标路径的权限类型：
   - readonly → 拒绝写入
   - readwrite → 允许
   - restricted → 需要用户明确授权
3. 返回权限验证结果
```

## 三、安全写入流程（核心）

### 3.1 新建文件流程

```
┌─────────────────────────────────────────────────────────────┐
│ 步骤 1: 验证权限                                              │
│ - 检查目标路径是否在 readwrite 列表                           │
│ - 确认文件名符合命名约定                                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤 2: 准备临时文件                                          │
│ - 在 workspace 中创建临时目录: ~/ObsidianVault/AI-Memory/temp/obsidian/       │
│ - 写入内容到临时文件                                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤 3: 向用户展示                                            │
│ - 显示文件内容摘要                                            │
│ - 显示目标路径                                                │
│ - 询问: "是否确认写入到 Obsidian Vault?"                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    ┌─────────┴─────────┐
                    ↓                   ↓
               用户确认             用户拒绝
                    ↓                   ↓
┌─────────────────────────────────┐   清理临时文件
│ 步骤 4: 执行写入                 │
│ - 复制临时文件到 vault           │
│ - 删除临时文件                   │
│ - 返回成功提示                   │
└─────────────────────────────────┘
```

### 3.2 修改现有文件流程

```

                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤 4: 对比差异                                              │

                              ↓

                              ↓
                    ┌─────────┴─────────┐
                    ↓                   ↓

│ - 删除临时文件                   │
│ - 返回成功提示                   │
```

## 六、受限操作流程

### 6.1 删除文件（需要用户授权）

```
1. 检查权限 → 在 restricted.delete 列表中
2. 展示文件信息（路径、大小、修改时间）
3. 询问: "确认删除此文件？此操作不可恢复！"
4. 用户确认后执行，否则取消
```

### 6.2 移动文件（需要用户授权）

```
1. 检查权限 → 在 restricted.move 列表中
2. 使用 obsidian-cli move（自动更新 wikilinks）
3. 展示移动前后路径
4. 询问: "确认移动？"
5. 用户确认后执行，否则取消
```

### 6.3 创建新模板（需要用户授权）

```
1. 检查权限 → 在 restricted.create_template 列表中
2. 按照标准写入流程操作
3. 额外询问: "这是一个新模板，确认创建？"
```

## 七、错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 权限不足 | 拒绝操作，提示权限规则 |
| 文件不存在 | 读取时提示；写入时检查是否为新建 |
| 差异过大 | 提示变化很大，建议用户手动审核 |
| vault 路径无效 | 尝试 obsidian-cli，失败则报错 |
| 临时文件失败 | 清理已创建的临时文件，提示错误 |

## 八、日常使用场景

### 场景 1: 保存对话日志

```
用户: "保存这次讨论到 Session-Logs"

AI:
1. 检查: 0-Inbox/Session-Logs/ 在 readwrite 列表 ✓
2. 命名: 2026-03-25-记忆系统讨论.md (符合约定)
3. 临时文件: ~/ObsidianVault/AI-Memory/temp/obsidian/session-temp.md
4. 展示内容摘要
5. 等待确认
6. 写入并清理临时文件
```

### 场景 2: 更新 CLAUDE.md

```
用户: "更新 CLAUDE.md，添加新规则"

AI:
1. 检查: CLAUDE.md 在 readwrite 列表 ✓
2. 读取原文件
3. 临时文件写入新内容
4. diff 对比，展示差异
5. 等待确认
6. 替换并清理临时文件
```

### 场景 3: 搜索知识

```
用户: "搜索关于机器人的笔记"

AI:
1. 使用 obsidian-cli search "robot"
2. 展示匹配结果
3. 用户选择读取某个笔记
```

## 九、命令速查

```bash
# 获取 vault 路径
obsidian-cli print-default --path-only

# 搜索笔记名称
obsidian-cli search "关键词"

# 搜索笔记内容
obsidian-cli search-content "关键词"

# 安全移动文件（自动更新 wikilinks）
obsidian-cli move "旧路径/文件.md" "新路径/文件.md"

# 备份文件
cp ~/ObsidianVault/AI-Memory/文件.md ~/ObsidianVault/AI-Memory/文件.md.bak

# 对比差异
diff ~/ObsidianVault/AI-Memory/temp/obsidian/temp.md ~/ObsidianVault/AI-Memory/目标.md
```

## 十一、最佳实践

1. **永远不要跳过权限检查** - 即使是熟悉的操作
2. **展示差异而非完整文件** - 除非文件很小
3. **使用 obsidian-cli 移动文件** - 保持 wikilink 一致性
4. **定期备份** - 特别是 CLAUDE.md 和重要笔记
5. **只保存有价值的内容** - 不是所有对话都需要记录
6. **保持命名规范** - 方便检索和整理
